加载管理

加载管理工具会实时记录项目运行时期的加载/卸载情况。使用加载管理工具可以全面直观的了解：

  • 资源加载/释放的场景
  • 资源加载/释放的总次数
  • 资源加载花费的时间和帧数

下图展示了加载管理工具记录的数据样本：

有的人看到这个工具后，可能会眼前一亮：资源的时间过长是不是制作上有问题？资源加载或卸载的场景不是预期的场景是不是代码逻辑有问题？资源加载和卸载的频次很高，是不是该考虑调整资源的生命周期管理？对，没错！这个工具就是用来暴露这些问题的。

提示

加载管理工具可以使用 编辑器顶部的 xasset/Loadables 菜单直接打开。

使用说明​

如何使用加载管理工具？请参考以下说明：

工具栏​

  • Record: 开始/停止记录加载/卸载情况
  • Loads：当前加载列表显示的对象范围，Loads 为所有加载过的对象，另外还可以切换为 Current 当前持有的加载对象， 和 Unloads 所有卸载了的加载对象。
  • SearchField: 搜索加载对象，支持模糊搜索。
  • Script Play Mode：当前代码运行模式，另外可以通过设置修改。
  • Export to...: 导出加载管理数据，可以导出为文本，或者导出到预定义的分包配置中。
  • Clear: 清除所有加载记录。
  • Frame: 帧数控制区域。

加载列表​

加载列表显示所有当前记录的数据，主要包含这些字段：

  • Path: 资源路径
  • Time: 加载时间和帧数
  • Size: 资源大小
  • LoadScene: 加载场景
  • UnloadScene: 卸载场景
  • Loads: 加载次数
  • Unloads: 卸载次数
  • Reference: 引用计数
提示

加载列表有助于帮助我们提前发现资源制作或使用的问题，例如，资源是否应该出现在加载场景，可以通过加载场景来判断；频繁加载或卸载的资源可以看下是否要调整生命周期；加载时间特别长的注意检查打包粒度或资源本身的规模。
